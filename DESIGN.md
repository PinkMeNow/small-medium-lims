## Layout & Spacing

- Base spacing unit: **4px** — use Tailwind `gap-*`, `p-*`, `px-*`, `space-y-*` on the 4/8px rhythm
- Field border width: **0px** (shadow provides depth instead)
- Field border radius: **12px** (`rounded-xl`)
- Surface/button border radius: **8px** (`rounded-lg`)

## Do's and Don'ts

**Do:**
- Use semantic tokens (`bg-accent`, `text-muted`, `border-border`) — never raw hex in component code
- Use the same token name in both light and dark mode; CSS resolves the correct value
- Keep spacing on the 4px/8px rhythm consistently
- Reserve `accent` for primary actions, `success`/`warning`/`danger` for their semantic meanings only
- Use surface levels, spacing, and type scale to create hierarchy before adding borders or decoration
- Keep descriptions short and scannable (2–3 lines max)
- Use `--chart-3` (= accent) as the primary chart series

**Don't:**
- Copy raw hex, shadow, radius, or spacing values into component code when a token exists
- Use `warning` for neutral decoration — only for genuine caution
- Stack extra custom shadows on Card or overlay components that already have elevation
- Use ALL CAPS labels or verbose headings
- Add hover/transition to non-interactive content
- Let badges, chips, or tags stretch full width — they should be content-sized
